### 3.2 详细接口定义示例

#### 接口1：用户注册

| 字段 | 内容 |
|------|------|
| 接口名称 | 用户注册 |
| 接口URL | `/api/register` |
| 请求方法 | POST |
| 请求头 | `Content-Type: application/json` |

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，3-20字符 |
| password | string | 是 | 密码，6-20字符 |

**请求示例**：
```json
{
    "username": "testuser",
    "password": "123456"
}
```

**响应参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码，200表示成功 |
| message | string | 提示信息 |
| data | object | 用户信息 |

**成功响应示例**：
```json
{
    "code": 200,
    "message": "注册成功",
    "data": {
        "id": 1,
        "username": "testuser",
        "role": "user"
    }
}
```

**失败响应示例**：
```json
{
    "code": 400,
    "message": "用户名已存在"
}
```

**错误码**：

| 错误码 | 含义 |
|--------|------|
| 400 | 参数错误/用户名已存在 |
| 500 | 服务器内部错误 |
