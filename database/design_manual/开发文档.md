## 一、项目概述

| 用户类型 | 描述 |
|----------|------|
| 图书爱好者 | 喜欢阅读，经常购书 |
| 学生 | 购买教材、课外书 |
| 网络文学爱好者 | 购买网络小说实体书 |
| 书店管理员 | 管理图书、订单 |
| 普通购书用户 | 一般购书需求 |

### 1.2 核心功能清单

#### 用户模块

| 功能 | 描述 | 说明 |
|------|------|------|
| 注册 | 用户名 + 密码 + 确认密码 | 无需邮箱/手机验证 |
| 登录 | 用户名 + 密码登录 | 登录后才能浏览和下单 |
| 退出登录 | 清除登录状态 | - |

#### 图书模块

| 功能 | 描述 | 说明 |
|------|------|------|
| 首页图书列表 | 分页展示所有图书（封面、书名、作者、价格） | 按上架时间倒序 |
| 搜索 | 按书名搜索 | 一个搜索框即可 |

#### 购物车模块

| 功能 | 描述 | 说明 |
|------|------|------|
| 加入购物车 | 从详情页加入，默认数量1 | 若已存在则数量+1 |
| 查看购物车 | 列表展示商品、单价、数量、小计 | 底部显示总金额 |

| 删除商品 | 单条删除 | - |
| 清空购物车 | 一键清空（需确认） | 可选 |

#### 订单与模拟支付模块

| 功能 | 描述 | 说明 |
|------|------|------|
| 去结算 | 从购物车跳转到确认页，用户填写收货地址和手机号 | 文本框手动填写，不做地址簿 |

| 确认收货 | 用户收到货后点击"确认收货" | 状态变为已完成 |
| 取消订单 | 仅待支付状态可取消，恢复库存 | - |

| 图书列表 | 分页展示所有图书 | 可搜索 |

| 编辑图书 | 修改以上所有字段 | - |

## 四、数据库设计

### 4.1 建表SQL

保存为 `docs/database.sql`

```sql
-- 1. 用户表
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(100) NOT NULL,
    role VARCHAR(20) DEFAULT 'user',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 2. 分类表
CREATE TABLE categories (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL
);

-- 3. 图书表
CREATE TABLE books (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    author VARCHAR(100) NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    stock INT NOT NULL DEFAULT 0,
    category_id INT REFERENCES categories(id),
    description TEXT,
    cover_url VARCHAR(500),
    status SMALLINT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 4. 购物车表
CREATE TABLE cart (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    book_id INT REFERENCES books(id) ON DELETE CASCADE,
    quantity INT NOT NULL DEFAULT 1,
    UNIQUE(user_id, book_id)
);

-- 5. 订单表
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    order_no VARCHAR(32) UNIQUE NOT NULL,
    total_amount DECIMAL(10,2) NOT NULL,
    address VARCHAR(200) NOT NULL,
    phone VARCHAR(20) NOT NULL,
    status SMALLINT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 6. 订单项表
CREATE TABLE order_items (
    id SERIAL PRIMARY KEY,
    order_id INT REFERENCES orders(id) ON DELETE CASCADE,
    book_id INT REFERENCES books(id),
    quantity INT NOT NULL,
    price DECIMAL(10,2) NOT NULL
);

-- 初始化分类数据
INSERT INTO categories (name) VALUES ('文学'), ('科技'), ('少儿'), ('教育'), ('网络文学');

-- 插入管理员账号（密码: admin123）
INSERT INTO users (username, password, role) VALUES ('admin', 'admin123', 'admin');
```

### 4.2 表关系说明

| 表名 | 外键 | 关联表 |
|------|------|--------|
| books | category_id | categories |
| cart | user_id, book_id | users, books |
| orders | user_id | users |
| order_items | order_id, book_id | orders, books |

## 七、API接口设计

### 7.1 统一响应格式

**成功响应**：
```json
{
    "code": 200,
    "message": "success",
    "data": {}
}
```

**失败响应**：
```json
{
    "code": 400,
    "message": "错误信息"
}
```

**未登录响应**：
```json
{
    "code": 401,
    "message": "请先登录"
}
```

**无权限响应**：
```json
{
    "code": 403,
    "message": "无权限访问"
}
```

## 九、关键业务逻辑

### 9.1 登录流程

```
1. 前端收集 username + password
2. POST /api/login
3. 后端查询 users 表
4. 比对密码（明文）
5. 正确 → 设置 Session（user_id, username, role）→ 返回成功
6. 前端保存用户信息到 localStorage → 跳转首页
```

### 9.2 下单流程（事务）

```
1. 前端提交 address + phone
2. POST /api/orders
3. 后端开启事务：
   a. 遍历购物车商品
   b. 检查库存是否充足
   c. 扣减库存（UPDATE books SET stock = stock - quantity）
   d. 生成订单号（yyyyMMdd + 6位随机数）
   e. 插入 orders 表（status=0）
   f. 插入 order_items 表（快照价格）
   g. 清空该用户的购物车
4. 提交事务
5. 返回订单ID
```

### 9.3 订单状态流转

| 当前状态 | 操作 | 新状态 | 额外动作 |
|----------|------|--------|----------|
| 待支付(0) | 模拟支付 | 待发货(1) | 无 |
| 待支付(0) | 取消订单 | 已取消(3) | 恢复库存 |
| 待发货(1) | 管理员发货 | 已完成(2) | 无 |
| 待发货(1) | 用户确认收货 | 已完成(2) | 无 |
